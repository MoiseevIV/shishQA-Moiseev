Все сйты вактически находятся на отдельном устроистве. Эти устроиства имеют отдельный номер IP. Благодаря этому номер можно найти источник ресурса. Если IP это номер , то DNS это телефонная книга. DNS - система доменных имен, которая обеспечивает связь между именем сайта и его цифровым номером. А если точнее, то когда человек вводит адрес сайта, то DNS конвертирует его в IP-адрес и передаёт на компьютер который поддерживает сайт. Он обрабатывает инфу и передаёт ей обратно для открытия сайта. Протокол HTTP и HTTPS используются для передачи данных в веб-браузерах. Они работают как почтовые службы. Разница между протоколами в том, что HTTPS защищёный инстурмент передачи, а HTTP нет. По факту он обладает встроенной уязвимостью. У HTTP две версии 1.1 и 2.0. Как я понял сейчас пока все пользуются версией 1.1. Версия 2.0 постепенно заходит на наши компьютеры. Простыми слвоами вторая версия работает куда быстрее. Вторая версия раставляет и управляет приоритетами запросов/потоков (клиент самостоятельно задаёт приоритет для сервера), сжимает заголовки, сервер самостоятельно может отправлять данные для клиента, которые тот еще не запросил, мультиплексирование потоков ( не требует несколько открытых TCP соединений). Версии HTTPS я не нашёл. Но я так понимаю что это протокол HTTP который получил сертификат безопасности. Есть несколько видов сертификатов: Domain validated (данные сервера защищены), Extended validation (домен пренадлежит компании), Organization validated (арантируют, что домен принадлежит организации).
